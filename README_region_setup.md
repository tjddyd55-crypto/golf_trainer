# 영역 설정 가이드 (Region Setup Guide)

## 문제 상황
- 숫자가 바뀔 때 (예: 800 rpm → 10,000 rpm) 가운데 정렬로 인해 위치가 틀어짐
- 좁은 영역으로는 숫자만 잡으면 단위가 영역 밖으로 나가서 OCR이 불안정

## 해결 방법
**영역을 넓게 잡아서 부호 + 숫자 + 단위 전체를 포함하도록 설정**

### 영역 설정 단계

1. `calibrate_regions.py` 실행
```bash
python calibrate_regions.py
```

2. 각 항목별로 영역 선택 시:
   - ✅ **올바른 방법**: 부호 + 숫자 + 단위 전체를 포함
     - 예: "L 3.0°" 전체
     - 예: "10,000 rpm" 전체
     - 예: "47.55 m/s" 전체
   
   - ❌ **잘못된 방법**: 숫자만 선택
     - 예: "3.0"만 선택
     - 예: "10,000"만 선택

3. 영역 선택 팁:
   - 숫자가 작을 때(800)와 클 때(10,000) 모두 포함되도록 넓게 잡기
   - 단위(rpm, m/s, ° 등)까지 포함
   - 좌우 여유 공간도 약간 포함

### 예시 영역 크기
- **좁은 영역 (문제)**: 숫자만 → 숫자가 바뀌면 위치 틀어짐
- **넓은 영역 (해결)**: "L 3.0°" 전체 → 숫자가 바뀌어도 안정적

## OCR 처리 방식
영역을 넓게 잡아도 걱정하지 마세요!
- OCR로 전체 텍스트 읽기 (예: "L 3.0°", "10,000 rpm")
- 정규표현식으로 숫자와 부호만 추출
- 단위는 자동으로 무시됨

## 개선된 parse_value 함수
- 쉼표 포함 숫자 처리: "10,000" → 10000
- 부호 인식 개선: "L 3.0°" → -3.0, "R 5.31 m" → 5.31
- 음수 처리: "-866 rpm" → -866
